ü§î Why Compare Languages?

Learning Benefits

  • Abstraction levels: See same concept at different granularities
  • Pattern recognition: Identify universal algorithmic patterns
  • Trade-off awareness: Understand performance vs convenience
  • Interview preparation: Many roles require multiple languages

The Three Perspectives

  • Pseudocode: Focus on logic, ignore syntax
  • C: Explicit memory, pointers, manual control
  • Python: High-level abstractions, built-in structures
Key Insight: The algorithm is the same‚Äîonly the expression differs. Understanding one helps understand all.

typedef struct TreeNode {
    int data;
    struct TreeNode *left;
    struct TreeNode *right;

TreeNode* create_node(int value) {

üìè Tree Height Calculation

üìù Pseudocode

FUNCTION Height(node):
    IF node = NULL THEN
        RETURN -1
    END IF
    
    leftHeight ‚Üê Height(node.left)
    rightHeight ‚Üê Height(node.right)
    
    RETURN 1 + MAX(leftHeight, 
                   rightHeight)
END FUNCTION

‚öôÔ∏è C

int tree_height(TreeNode *node) {
    if (node == NULL) {
        return -1;
    }
    
    int left_h = tree_height(node->left);
    int right_h = tree_height(node->right);
    
    int max = (left_h > right_h) ? 
               left_h : right_h;
    return 1 + max;
}

üêç Python

def height(node):
    if node is None:
        return -1
    
    left_h = height(node.left)
    right_h = height(node.right)
    
    return 1 + max(left_h, right_h)


# One-liner version:
def height(n):
    return -1 if n is None else \
           1 + max(height(n.left), 
                   height(n.right))

üóëÔ∏è Memory Management / Deletion

üìù Pseudocode

FUNCTION FreeTree(node):
    IF node = NULL THEN
        RETURN
    END IF
    
    // MUST free children first!
    FreeTree(node.left)
    FreeTree(node.right)
    
    // Then free current node
    DEALLOCATE(node)
END FUNCTION

// This is POSTORDER traversal
// Children before parent!

‚öôÔ∏è C

void free_tree(TreeNode *node) {
    if (node == NULL) {
        return;
    }
    
    // Free children FIRST
    free_tree(node->left);
    free_tree(node->right);
    
    // Then free this node
    free(node);
}

// Usage:
// free_tree(root);
// root = NULL;  // Prevent dangling

üêç Python

# Python: NO EXPLICIT FREEING NEEDED!

# Garbage collector handles it
# When no references exist, memory freed

# To "delete" a tree:
root = None  # That's it!

# Or for explicit deletion:
def delete_tree(node):
    """Optional: explicitly break refs"""
    if node is None:
        return
    delete_tree(node.left)
    delete_tree(node.right)
    node.left = None
    node.right = None
    # GC will clean up
Critical C Insight: Memory leaks are the #1 tree-related bug. Always use postorder for deletion!

üßÆ Expression Tree Evaluation

üìù Pseudocode

FUNCTION Evaluate(node):
    IF IsOperand(node) THEN
        RETURN node.value
    END IF
    
    left ‚Üê Evaluate(node.left)
    right ‚Üê Evaluate(node.right)
    
    SWITCH node.operator:
        CASE '+': RETURN left + right
        CASE '-': RETURN left - right
        CASE '*': RETURN left * right
        CASE '/': RETURN left / right
    END SWITCH
END FUNCTION

‚öôÔ∏è C

int evaluate(ExprNode *node) {
    if (!node->is_operator) {
        return node->value;
    }
    
    int left = evaluate(node->left);
    int right = evaluate(node->right);
    
    switch (node->op) {
        case '+': return left + right;
        case '-': return left - right;
        case '*': return left * right;
        case '/': return left / right;
        default:  return 0;
    }
}

üêç Python

def evaluate(node):
    if not node.is_operator:
        return node.value
    
    left = evaluate(node.left)
    right = evaluate(node.right)
    
    ops = {
        '+': lambda a, b: a + b,
        '-': lambda a, b: a - b,
        '*': lambda a, b: a * b,
        '/': lambda a, b: a // b
    }
    return ops[node.op](left, right)

üìä Level-Order Traversal (BFS)

üìù Pseudocode

FUNCTION LevelOrder(root):
    IF root = NULL THEN
        RETURN
    END IF
    
    queue ‚Üê NEW Queue
    ENQUEUE(queue, root)
    
    WHILE NOT IsEmpty(queue):
        node ‚Üê DEQUEUE(queue)
        PROCESS(node.data)
        
        IF node.left ‚â† NULL THEN
            ENQUEUE(queue, node.left)
        END IF
        IF node.right ‚â† NULL THEN
            ENQUEUE(queue, node.right)
        END IF
    END WHILE
END FUNCTION

‚öôÔ∏è C

void level_order(TreeNode *root) {
    if (root == NULL) return;
    
    Queue *q = queue_create();
    queue_enqueue(q, root);
    
    while (!queue_is_empty(q)) {
        TreeNode *n = queue_dequeue(q);
        printf("%d ", n->data);
        
        if (n->left != NULL)
            queue_enqueue(q, n->left);
        if (n->right != NULL)
            queue_enqueue(q, n->right);
    }
    queue_destroy(q);
}
// Requires Queue implementation!

üêç Python

from collections import deque

def level_order(root):
    if root is None:
        return
    
    queue = deque([root])
    
    while queue:
        node = queue.popleft()
        print(node.data, end=' ')
        
        if node.left:
            queue.append(node.left)
        if node.right:
            queue.append(node.right)

# Return as list of levels:
def level_order_levels(root):
    if not root:
        return []
    result, queue = [], deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):
            n = queue.popleft()
            level.append(n.data)
            if n.left: queue.append(n.left)
            if n.right: queue.append(n.right)
        result.append(level)
    return result

üí° Key Insights

Universal Patterns

  • Recursion structure is identical across all languages
  • Base case + recursive case pattern always applies
  • Traversal order (pre/in/post) concept is language-agnostic
  • Time/space complexity is the same regardless of implementation

Language-Specific Concerns

  • C: Manual memory is both power and responsibility
  • Python: Convenience hides important details
  • Pseudocode: Great for design, useless for execution
  • All: Correctness first, then optimise
The Master Principle: Learn algorithms in pseudocode, implement in your target language, optimise based on profiling. The algorithm is eternal; implementations are temporal.
