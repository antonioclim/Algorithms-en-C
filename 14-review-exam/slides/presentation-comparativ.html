ü§î Why Compare Languages?

Understanding Abstraction Levels

  • Pseudocode ‚Äî Pure algorithmic logic, language-agnostic
  • C ‚Äî Low-level control, manual memory, explicit types
  • Python ‚Äî High-level abstraction, automatic memory, dynamic types

Key Insight

The same algorithm can look very different across languages whilst maintaining identical logic and complexity.

Benefits of Comparison

  • Separates algorithm from implementation details
  • Reveals language-specific idioms
  • Highlights trade-offs (verbosity vs clarity)
  • Improves translation skills between languages
  • Deepens understanding of underlying concepts

üîç Binary Search

üìù Pseudocode
FUNCTION BinarySearch(A, target):
    left ‚Üê 0
    right ‚Üê length(A) - 1
    
    WHILE left ‚â§ right:
        mid ‚Üê left + (right - left) / 2
        
        IF A[mid] = target:
            RETURN mid
        ELSE IF A[mid] < target:
            left ‚Üê mid + 1
        ELSE:
            right ‚Üê mid - 1
    
    RETURN -1  // Not found
‚öôÔ∏è C
int binary_search(int *arr, int n, 
                  int target) {
    int left = 0;
    int right = n - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        
        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    
    return -1;
}

‚ö° Quick Sort

üìù Pseudocode
FUNCTION QuickSort(A, low, high):
    IF low < high:
        pivot ‚Üê Partition(A, low, high)
        QuickSort(A, low, pivot - 1)
        QuickSort(A, pivot + 1, high)

FUNCTION Partition(A, low, high):
    pivot ‚Üê A[high]
    i ‚Üê low - 1
    
    FOR j ‚Üê low TO high - 1:
        IF A[j] ‚â§ pivot:
            i ‚Üê i + 1
            SWAP(A[i], A[j])
    
    SWAP(A[i + 1], A[high])
    RETURN i + 1
‚öôÔ∏è C
void swap(int *a, int *b) {
    int t = *a; *a = *b; *b = t;
}

int partition(int *arr, int low, int high) {
    int pivot = arr[high];
    int i = low - 1;
    
    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(&arr[i], &arr[j]);
        }
    }
    swap(&arr[i + 1], &arr[high]);
    return i + 1;
}

void quicksort(int *arr, int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quicksort(arr, low, pi - 1);
        quicksort(arr, pi + 1, high);
    }
}
üêç Python
def quicksort(arr):
    if len(arr) <= 1:
        return arr
    
    pivot = arr[-1]
    left = [x for x in arr[:-1] 
            if x <= pivot]
    right = [x for x in arr[:-1] 
             if x > pivot]
    
    return quicksort(left) + [pivot] \
           + quicksort(right)

üõ§Ô∏è Dijkstra's Shortest Path

üìù Pseudocode
FUNCTION Dijkstra(G, source):
    FOR each vertex v in G:
        dist[v] ‚Üê INFINITY
        prev[v] ‚Üê UNDEFINED
    dist[source] ‚Üê 0
    Q ‚Üê priority queue of all vertices
    
    WHILE Q is not empty:
        u ‚Üê vertex with min dist in Q
        REMOVE u from Q
        
        FOR each neighbour v of u:
            alt ‚Üê dist[u] + weight(u, v)
            IF alt < dist[v]:
                dist[v] ‚Üê alt
                prev[v] ‚Üê u
                DECREASE-KEY(Q, v, alt)
    
    RETURN dist, prev
‚öôÔ∏è C
void dijkstra(Graph *g, int src, 
              int *dist, int *prev) {
    MinHeap *pq = heap_create(g->V);
    
    for (int v = 0; v < g->V; v++) {
        dist[v] = INT_MAX;
        prev[v] = -1;
    }
    dist[src] = 0;
    heap_insert(pq, src, 0);
    
    while (!heap_empty(pq)) {
        int u = heap_extract_min(pq);
        
        for (Edge *e = g->adj[u]; e; 
             e = e->next) {
            int v = e->dest;
            if (dist[u] + e->weight < dist[v]) {
                dist[v] = dist[u] + e->weight;
                prev[v] = u;
                heap_decrease(pq, v, dist[v]);
            }
        }
    }
    heap_destroy(pq);
}
üêç Python
import heapq

def dijkstra(graph, src):
    dist = {v: float('inf') 
            for v in graph}
    dist[src] = 0
    prev = {v: None for v in graph}
    pq = [(0, src)]
    
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]:
            continue
            
        for v, weight in graph[u]:
            if dist[u] + weight < dist[v]:
                dist[v] = dist[u] + weight
                prev[v] = u
                heapq.heappush(pq, 
                    (dist[v], v))
    
    return dist, prev

üî¢ Dynamic Programming: Fibonacci

üìù Pseudocode
// Naive recursive (exponential)
FUNCTION Fib_Naive(n):
    IF n ‚â§ 1:
        RETURN n
    RETURN Fib(n-1) + Fib(n-2)

// DP with memoisation
FUNCTION Fib_Memo(n, memo):
    IF n ‚â§ 1:
        RETURN n
    IF memo[n] is defined:
        RETURN memo[n]
    memo[n] ‚Üê Fib(n-1) + Fib(n-2)
    RETURN memo[n]

// DP with tabulation
FUNCTION Fib_Tab(n):
    IF n ‚â§ 1: RETURN n
    prev, curr ‚Üê 0, 1
    FOR i ‚Üê 2 TO n:
        prev, curr ‚Üê curr, prev+curr
    RETURN curr
‚öôÔ∏è C
// Space-optimised O(1)
long long fib(int n) {
    if (n <= 1) return n;
    
    long long prev = 0;
    long long curr = 1;
    
    for (int i = 2; i <= n; i++) {
        long long next = prev + curr;
        prev = curr;
        curr = next;
    }
    
    return curr;
}

// With memoisation
long long memo[100] = {0};
long long fib_memo(int n) {
    if (n <= 1) return n;
    if (memo[n]) return memo[n];
    return memo[n] = fib_memo(n-1) 
                   + fib_memo(n-2);
}

    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a

üìè Longest Common Subsequence

üìù Pseudocode
FUNCTION LCS(X, Y):
    m ‚Üê length(X)
    n ‚Üê length(Y)
    dp ‚Üê 2D array [m+1][n+1]
    
    FOR i ‚Üê 0 TO m:
        FOR j ‚Üê 0 TO n:
            IF i = 0 OR j = 0:
                dp[i][j] ‚Üê 0
            ELSE IF X[i-1] = Y[j-1]:
                dp[i][j] ‚Üê dp[i-1][j-1] + 1
            ELSE:
                dp[i][j] ‚Üê MAX(dp[i-1][j],
                               dp[i][j-1])
    
    RETURN dp[m][n]
‚öôÔ∏è C
int lcs(const char *X, const char *Y) {
    int m = strlen(X);
    int n = strlen(Y);
    
    int **dp = malloc((m+1) * sizeof(int*));
    for (int i = 0; i <= m; i++)
        dp[i] = calloc(n+1, sizeof(int));
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (X[i-1] == Y[j-1])
                dp[i][j] = dp[i-1][j-1] + 1;
            else
                dp[i][j] = (dp[i-1][j] > dp[i][j-1])
                         ? dp[i-1][j] : dp[i][j-1];
        }
    }
    
    int result = dp[m][n];
    // Free memory...
    return result;
}
üêç Python
def lcs(X, Y):
    m, n = len(X), len(Y)
    dp = [[0] * (n + 1) 
          for _ in range(m + 1)]
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j],
                               dp[i][j-1])
    
    return dp[m][n]

üîÑ Breadth-First Search

üìù Pseudocode
FUNCTION BFS(G, start):
    visited ‚Üê empty set
    queue ‚Üê empty queue
    
    ENQUEUE(queue, start)
    ADD start TO visited
    
    WHILE queue is not empty:
        vertex ‚Üê DEQUEUE(queue)
        PROCESS(vertex)
        
        FOR each neighbour of vertex:
            IF neighbour NOT IN visited:
                ADD neighbour TO visited
                ENQUEUE(queue, neighbour)

// Complexity: O(V + E)

üêç Python
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    result = []
    
    while queue:
        vertex = queue.popleft()
        result.append(vertex)
        
        for neighbour in graph[vertex]:
            if neighbour not in visited:
                visited.add(neighbour)
                queue.append(neighbour)
    
    return result

üìù Pseudocode
FUNCTION Hash(key, size):
    hash ‚Üê 0
    FOR each char c IN key:
        hash ‚Üê hash * 33 + c
    RETURN hash MOD size

FUNCTION Insert(table, key, value):
    index ‚Üê Hash(key, table.size)
    // Handle collision (chaining)
    ADD (key, value) TO table[index]

FUNCTION Get(table, key):
    index ‚Üê Hash(key, table.size)
    FOR each (k, v) IN table[index]:
        IF k = key:
            RETURN v
    RETURN NULL
‚öôÔ∏è C
typedef struct Node {
    char *key;
    void *value;
    struct Node *next;
} Node;

size_t hash(const char *key, size_t sz) {
    size_t h = 5381;
    int c;
    while ((c = *key++))
        h = ((h << 5) + h) + c;
    return h % sz;
}

void *ht_get(HashTable *ht, 
             const char *key) {
    size_t idx = hash(key, ht->size);
    for (Node *n = ht->buckets[idx]; 
         n; n = n->next)
        if (strcmp(n->key, key) == 0)
            return n->value;
    return NULL;
}

# For learning, manual impl:
class HashTable:
    def __init__(self, size=100):
        self.size = size
        self.table = [[] for _ in range(size)]
    
    def _hash(self, key):
        return hash(key) % self.size
    
    def set(self, key, value):
        idx = self._hash(key)
        for pair in self.table[idx]:
            if pair[0] == key:
                pair[1] = value
                return
        self.table[idx].append([key, value])

üí° Key Insights

Algorithm Invariants

  • Time complexity remains constant across languages
  • Space complexity may vary due to overhead
  • Correctness depends on algorithm, not language
  • Logic flow is identical when properly translated

Language-Specific Considerations

  • C: Pointer arithmetic, manual bounds checking
  • Python: List comprehensions, generator expressions
  • Both: Standard library functions are optimised

‚úì Remember

The best programmers understand algorithms at the pseudocode level and can implement them in multiple languages. Focus on the algorithm first, then adapt to the language's idioms.

Conclusion

Understanding algorithms across languages deepens comprehension and improves implementation skills.
